# architect-mindset

Behavior skill for design work: boundaries, contracts, tradeoffs, and specs
that survive being executed by someone with less context than their author.

Part of the behavior family:

- [execution-mindset](../execution-mindset/) — how to operate on any task (always on)
- **architect-mindset** — how to behave when the deliverable is a design
- [code-engineering](../code-engineering/) — how to behave when the deliverable is code

Core ideas: model the forces before proposing structure; boundaries are
promises with violation signals; design for the process dying mid-step;
classify decisions by cost of reversal; enumerate consumers before changing
anything shared; write specs (role / what / how) for the cheapest executor.
